# Serenity BDD 5.3.4 Release Notes

**Playwright Screenshot and HTML Source Permission Fixes**

This release fixes excessive screenshot and HTML source artifact generation in Playwright tests by ensuring the `PlaywrightStepListener` and `BrowseTheWebWithPlaywright` respect the `serenity.take.screenshots` and `serenity.store.html` configuration properties.

## Bug Fixes

### PlaywrightStepListener Now Respects Screenshot Permissions

Previously, the `PlaywrightStepListener` captured screenshots unconditionally at every step boundary, generating 100+ artifacts even when `serenity.take.screenshots=FOR_FAILURES` was configured. The listener now checks `ScreenshotPermission` before capturing, matching the behavior of `BaseStepListener`:

- **FOR_FAILURES** — screenshots are only taken on step failure
- **AFTER_EACH_STEP** — screenshots are taken after every step
- **DISABLED** — no screenshots are captured

### PlaywrightStepListener Now Respects SERENITY_STORE_HTML

HTML page source capture in the step listener now checks the `serenity.store.html` property:

- **NEVER** — page source is never recorded
- **ALWAYS** — page source is always recorded
- **FAILURES** (default) — page source is only recorded when the current test has failed

### BrowseTheWebWithPlaywright Conditional HTML Source Recording

The `takeScreenShot()` method in `BrowseTheWebWithPlaywright` now conditionally records HTML source based on `serenity.store.html`, following the same pattern used by `SoundEngineer` in the WebDriver module.

### Photographer Directory Creation Fix

`Photographer.takesAScreenshot()` now ensures the output directory exists before writing screenshot files, preventing `IOException` when tests run in certain execution orders.

## Affected Components

| Component | Change |
|-----------|--------|
| `PlaywrightStepListener` | Screenshot permission checks via `ScreenshotPermission`; HTML source gated by `SERENITY_STORE_HTML` |
| `BrowseTheWebWithPlaywright` | `takeScreenShot()` conditionally records HTML source based on `SERENITY_STORE_HTML` |
| `Photographer` | Added `Files.createDirectories()` before writing screenshot files |

## Upgrade

```xml
<dependency>
    <groupId>net.serenity-bdd</groupId>
    <artifactId>serenity-core</artifactId>
    <version>5.3.4</version>
</dependency>
```

No code changes are required. This is a drop-in replacement for 5.3.3.
