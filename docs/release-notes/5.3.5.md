# Serenity BDD 5.3.5 Release Notes

**Soft Assertion Fix, Screenshot Permission Hardening, and Playwright Context Switching Documentation**

This release fixes a critical soft assertion bug where RuntimeExceptions were incorrectly swallowed, closes remaining screenshot permission gaps in the Playwright step listener, and adds comprehensive tests and documentation for iframe and multi-page interactions.

## Bug Fixes

### RuntimeException Not Treated as Hard Failure When Soft Assertions Enabled (#3727)

When soft assertions were active via `Ensure.enableSoftAssertions()`, runtime exceptions thrown during test steps (e.g., `NoSuchElementException`, `NullPointerException`) were incorrectly captured as soft assertion failures instead of terminating the test. This caused:

- Tests continuing to execute despite critical infrastructure failures
- Cascading failures that obscured the root cause
- Extended test execution time from unnecessary subsequent steps

**Now:** Only `AssertionError` subclasses are treated as soft failures. Runtime exceptions and checked exceptions always propagate immediately, terminating the test even when soft assertions are active.

### PlaywrightStepListener takeScreenshots() Permission Checks

The `takeScreenshots()` interface methods in `PlaywrightStepListener` were called by the framework on every step completion without checking the `serenity.take.screenshots` configuration, writing screenshot and HTML source files unconditionally. Additionally, the replay variants (`stepFinished(List)`, `stepFailed(_, List)`) were re-capturing screenshots that had already been written during `takeScreenshots()`, causing double file writes in the Cucumber session path.

**Now:** All screenshot capture paths in `PlaywrightStepListener` check `ScreenshotPermission` before writing files, and replay variants no longer re-capture.

## New Documentation

### Playwright Context Switching Guide (#3725)

Added comprehensive documentation and tests for working with multiple pages and iframes:

- **`SwitchTo`** — Managing multiple browser pages/tabs (open, switch by index/title/URL, close)
- **`WithinFrame`** — Interacting with elements inside iframes (click, fill, type, clear, custom actions)
- **Nested frames** — Using `Target.inFrame()` chains for deeply nested iframes

## Affected Components

| Component | Change |
|-----------|--------|
| `StepInterceptor` | Only suppress `AssertionError` during soft assertions; re-throw all other exceptions |
| `PlaywrightStepListener` | Permission checks on `takeScreenshots()` methods; no-op replay variants to prevent double capture |
| `serenity-screenplay-playwright/README.md` | New documentation for `SwitchTo` and `WithinFrame` |

## Upgrade

```xml
<dependency>
    <groupId>net.serenity-bdd</groupId>
    <artifactId>serenity-core</artifactId>
    <version>5.3.5</version>
</dependency>
```

No code changes are required. This is a drop-in replacement for 5.3.4.
